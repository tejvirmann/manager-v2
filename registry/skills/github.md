The GitHub board at `Tejvir-PM-Stack/manager` is the primary system for tracking all work.

## Issue operations
- Create issues for tasks, features, bugs, and project items
- Update title, body, labels, assignees, milestone
- Close issues when work is confirmed done
- Add comments for updates or notes
- Always confirm the action you took after any GitHub operation

## Board views
- **Board view**: use the status field to move items between columns (Todo, In Progress, Done)
- **Roadmap/timeline view**: use start date and target date fields on project items

## Search and query
- Filter issues by label, assignee, state, or milestone
- When asked "what's on my plate": list open issues grouped by priority or label
- For daily briefings: show open issues sorted by urgency, highlight anything blocked or overdue
